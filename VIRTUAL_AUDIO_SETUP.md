# Настройка виртуального аудио канала

Для передачи музыки с приложений (Яндекс.Музыка, Spotify и т.д.) без локального воспроизведения звука.

## Windows

### VB-Audio Virtual Cable

1. **Скачайте VB-Audio Virtual Cable:**
   - Официальный сайт: https://vb-audio.com/Cable/
   - Скачайте и установите последнюю версию

2. **Настройка Windows:**
   - После установки в настройках звука появится "CABLE Input"
   - Это виртуальное устройство вывода

3. **Настройка приложения для музыки:**
   - Откройте Яндекс.Музыку, Spotify или другое приложение
   - В настройках выберите "CABLE Input" как устройство вывода
   - Теперь звук будет идти на виртуальный кабель, а не в колонки

4. **Настройка LynchFM Broadcaster:**
   - В приложении выберите источник звука: "Виртуальный канал"
   - В списке устройств выберите "CABLE Output"
   - Подключитесь к серверу и начните вещание
   - Звук будет транслироваться на радио, но не будет слышен локально

## macOS

### BlackHole

1. **Установите BlackHole:**
   ```bash
   brew install blackhole-16ch
   ```
   Или скачайте с: https://github.com/ExistentialAudio/BlackHole

2. **Создайте Multi-Output Device:**
   - Откройте "Аудио MIDI настройка" (Audio MIDI Setup)
   - Нажмите "+" и выберите "Создать устройство множественного вывода"
   - Добавьте ваши колонки/наушники и BlackHole
   - Сохраните как "Multi-Output"

3. **Настройка:**
   - В настройках macOS выберите "Multi-Output" как устройство вывода
   - В Broadcaster выберите "BlackHole" как источник

## Linux

### JACK Audio Connection Kit

1. **Установите JACK:**
   ```bash
   sudo apt install jackd2 qjackctl
   ```

2. **Настройка:**
   - Запустите QjackCtl
   - Настройте JACK сервер
   - Подключите приложение музыки к виртуальному выходу JACK
   - В Broadcaster выберите виртуальный вход JACK

## Альтернативный метод (Screen Capture)

Если виртуальный кабель не работает, используйте Screen Capture:

1. В Broadcaster выберите "Системный звук (Screen Capture)"
2. При запросе доступа выберите вкладку/окно с музыкой
3. Разрешите захват звука
4. Звук будет транслироваться, но будет слышен локально

## Важные замечания

- Виртуальный кабель - лучший вариант, так как звук НЕ будет слышен локально
- Screen Capture - альтернатива, но звук будет слышен
- Для каждого метода может потребоваться дополнительная настройка
